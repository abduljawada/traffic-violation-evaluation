# Contributing

Thank you for your interest in contributing! Here's how you can help:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/traffic-violation-detection.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Set up your environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

## Development Workflow

1. Make your changes
2. Test thoroughly: `python evaluate.py <test_submission.json>`
3. Commit with clear messages: `git commit -m "Add feature XYZ"`
4. Push to your fork: `git push origin feature/your-feature`
5. Submit a Pull Request

## Reporting Issues

- Check existing issues before opening a new one
- Include clear descriptions and reproduction steps
- Attach relevant submission files if applicable

## Code Style

- Follow PEP 8 guidelines
- Add docstrings to functions
- Include comments for complex logic
